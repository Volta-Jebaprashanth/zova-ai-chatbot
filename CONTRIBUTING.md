## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check the existing issues to avoid duplicates. When you create a bug report, include as many details as possible:

- **Use a clear and descriptive title**
- **Describe the exact steps to reproduce the problem**
- **Provide specific examples** to demonstrate the steps
- **Describe the behavior you observed** and what you expected to see
- **Include screenshots or GIFs** if applicable
- **Include your environment details** (browser, OS, etc.)

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, include:

- **Use a clear and descriptive title**
- **Provide a detailed description** of the suggested enhancement
- **Explain why this enhancement would be useful** to most Zova users
- **List some examples** of how it would be used

### Pull Requests

1. Fork the repository
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit your changes (`git commit -m 'Add some amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

#### Pull Request Guidelines

- **Keep it focused** - One feature or fix per PR
- **Write clear commit messages** - Explain what and why
- **Update documentation** - If you change functionality, update the README
- **Test your changes** - Make sure everything works
- **Follow the existing code style**

## Development Setup

1. Clone the repository
```bash
git clone https://github.com/Volta-Jebaprashanth/zova-ai-chatbot.git
cd zova-ai-chatbot
```

2. Test locally by integrating the chatbot folder into a test website

3. Make your changes and test thoroughly

## Code Style

- Use meaningful variable and function names
- Comment complex logic
- Keep functions focused and concise
- Follow consistent indentation (2 spaces)

## Configuration Changes

If you modify `config.json` structure:
- Update the README documentation
- Provide migration notes if breaking backward compatibility
- Update example configurations

## Testing

Before submitting a PR:
- Test the chatbot in multiple browsers (Chrome, Firefox, Safari)
- Test on both desktop and mobile devices
- Verify all configuration options work correctly
- Check that data sources load properly

## Questions?

Feel free to open an issue with the label `question` if you need help or clarification!

## Code of Conduct

This project follows a Code of Conduct. By participating, you are expected to uphold this code.
